# Release Notes - v1.0.0

Welcome to the official 1.0.0 release of **Task Manager**. This version represents a complete, production-ready internal ticketing system with a robust backend, interactive frontend, and comprehensive management features.

## 🚀 Project Architecture

Task Manager is built following **Clean Architecture** principles, ensuring separation of concerns, testability, and maintainability.

- **Frontend:** React 19 SPA (Single Page Application) using TypeScript.
- **Backend:** .NET 10 (C# 14) Web API.
- **Database:** PostgreSQL 17.
- **Layers:**
  - **Core:** Domain entities and business logic (No dependencies).
  - **Infrastructure:** Persistence (EF Core), Identity, and external services.
  - **API:** RESTful endpoints, DTOs, and OpenAPI documentation.

## 🛠 Technologies Used

### Frontend
- **React 19 & TypeScript:** Modern UI development.
- **Vite:** Fast build tool and dev server.
- **Tailwind CSS:** Utility-first styling for a polished, responsive design.
- **Zustand:** Lightweight state management.
- **TanStack Query (v5):** Efficient server state synchronization and caching.
- **React Router 7:** Advanced routing and navigation.

### Backend
- **.NET 10 & C# 14:** High-performance backend framework.
- **Entity Framework Core:** ORM for PostgreSQL.
- **ASP.NET Core Identity:** Secure user authentication and Role-Based Access Control (RBAC).
- **JWT (JSON Web Tokens):** Secure, stateless authentication.
- **Scalar:** Interactive API documentation (OpenAPI/Swagger successor).

### Infrastructure & CI/CD
- **Docker & Docker Compose:** Containerization for consistent environments.
- **GitHub Actions:** Automated CI pipeline for building and testing.
- **PostgreSQL 17:** Relational database for persistent storage.

## 👷 CI Pipelines

Our **GitHub Actions** pipeline (`ci.yml`) ensures code quality and stability through:
- **Build Validation:** Automatic builds for both Backend (.NET) and Frontend (Vite/React) on every push and pull request.
- **Automated Testing:** Execution of unit and integration tests for the backend solution.
- **Linting:** Verification of code standards and formatting.

## ✨ Features

- **Authentication & RBAC:** Secure login with distinct permissions for Admin, Technician, and Member roles.
- **Ticket Lifecycle Management:** Create, view, update, and resolve tickets with priority levels (Low, Medium, High).
- **User Management (Admin):** Full CRUD for user accounts, including role assignment and status toggling.
- **Department Management (Admin):** Manage organizational departments (HR, Tech, Marketing, etc.).
- **Analytics Dashboard (Admin):** Visualize resolution velocity, technician performance, and ticket trends with date filtering.
- **Help Center:** Integrated guide for users to learn how to navigate and use the system.
- **Modern UI/UX:** Responsive design with "Bento-lite" stats, clean tables, and intuitive modals.

## 🐛 Bug Fixes & Improvements

- **Database Auto-Migration:** Implemented runtime migrations to ensure the schema is always in sync with the code (Fixes missing 'Priority' column errors).
- **UI Contrast Fixes:** Improved readability of footer text and header profile information in dark/light modes.
- **Priority Icon Standardization:** Scaled "Medium" priority icons to match Low and High visibility.
- **Password Visibility:** Fixed the "eye" toggle logic in the login form.
- **Advanced Filtering:** Fixed logic for filtering tickets by Department and Priority.
- **Branding Update:** Standardized all UI labels from "Indigo Slate" to "Task Manager".
